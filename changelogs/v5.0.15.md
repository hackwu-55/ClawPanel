## v5.0.15 - QQ 通道停止修复 + IM 插件仓库 + 架构文档

### Bug 修复
- **fix**: 关闭 QQ (NapCat) 通道时，同时停止 NapCat 容器（Linux Docker）或进程（Windows Shell），彻底解决通道关闭后日志仍持续出现 WS 连接/断开刷屏问题
- **fix**: `NapCatMonitor` 新增 `Pause()` / `Resume()` 方法，关闭 QQ 通道时暂停监控轮询，启用时自动恢复，避免无效连接检测产生噪音日志

### 新功能
- **feat**: 插件仓库新增四个 IM 通道插件，用户可在插件中心一键安装：
  - 飞书 / Lark (`@openclaw/feishu`)
  - Discord (`@openclaw/discord`)
  - Telegram (`@openclaw/telegram`)
  - Slack (`@openclaw/slack`)
- **feat**: 插件安装支持 npm 全局包（`npmPackage` 字段），优先使用 npmmirror 镜像加速，安装失败自动 fallback 到官方源
- **feat**: 新增 QQ NapCat 通道诊断与修复脚本，三端适配：
  - Linux / macOS: `scripts/fix-qq-napcat.sh`
  - Windows: `scripts/fix-qq-napcat.ps1`
- **feat**: 新增架构说明文档 `docs/qq-napcat-guide.md`，详细介绍 NapCat + OpenClaw + ClawPanel 通讯机制、配置结构、常见问题与修复方法
