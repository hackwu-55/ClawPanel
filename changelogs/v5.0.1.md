# ClawPanel v5.0.1 — 配置自动检测 · NapCat 连接监控 · 活动日志增强

**发布日期**: 2026-02-24

## ✨ 新功能

### 🔍 OpenClaw/NapCat 配置自动检测 + 一键修复
- 新增「配置检测」Tab（系统配置页面），自动扫描 OpenClaw 和 NapCat 核心配置文件
- 检测项包括：
  - NapCat `onebot11.json`：WebSocket/HTTP 服务启用状态、端口配置、`reportSelfMessage` 设置
  - NapCat `webui.json`：WebUI Token 是否为空
  - OpenClaw `openclaw.json`：模型提供商 API Key、主模型配置、QQ 通道 WebSocket 地址
  - 端口可达性检测（3000/3001/6099）
- 每个问题显示严重级别（错误/警告/信息）、当前值与期望值、文件路径
- 支持单项修复和「一键修复全部」，修复后自动重启 NapCat 容器
- API：`GET /api/openclaw/config/check`、`POST /api/openclaw/config/fix`

### 📡 NapCat 掉线自动重连 + 实时状态监控
- 后台每 30 秒检测 NapCat 连接状态（容器运行、WS 连接、HTTP 可用、QQ 登录）
- 通道管理 → QQ 通道页面新增实时连接状态面板：
  - 状态指示灯（🟢 在线 / 🟡 重连中 / 🟠 登录失效 / 🔴 离线/停止）
  - 显示 QQ 昵称和号码
  - 容器/WS/HTTP/QQ 四项状态一览
- 自动重连：连续 3 次检测离线后自动执行 `docker restart openclaw-qq`
- 可配置重连次数上限（默认 10 次），防止无限重连
- 手动重连按钮 + 重连日志查看
- 状态变化通过 WebSocket 实时推送到前端
- API：`GET /api/napcat/status`、`POST /api/napcat/reconnect`、`GET /api/napcat/reconnect-logs`、`PUT /api/napcat/monitor-config`

### 📋 活动日志增强
- **Bot 回复消息现在正确显示**：支持 OneBot11 `message_sent` 事件类型
- **清除按钮修复**：点击垃圾桶图标现在同时清除前端列表和数据库记录

## 🐛 Bug 修复

- **修复活动日志不显示 Bot 回复**：NapCat 的 bot 自身消息使用 `post_type=message_sent`，之前只处理了 `message`
- **修复 NapCat 容器 entrypoint 每次重启覆写配置**：`napcat-entrypoint.sh` 不再无条件覆写 `onebot11.json`，改为仅在文件不存在时初始化
- **修复配置检测误报**：OpenClaw QQ 通道使用 `wsUrl` 而非 `gateway` 子对象
- **修复 NapCat WebUI 认证方式**：使用 SHA256 哈希认证（`sha256(token + ".napcat")`）而非明文 token
- **修复活动日志清除按钮无效**：`clearEvents` 现在同时清除前端状态和调用 API 清除数据库
- **修复 NapCat 监控误触发重启**：增加连续离线计数（需 3 次连续离线才触发重连），重连期间跳过检查
